# Examples

Topogram uses three example relationships:

- [generated](./generated): Topogram-owned generated reference apps
- [maintained](./maintained): Topogram-owned maintained proof apps
- [imported](./imported): imported brownfield proof targets managed through the separate demo/proof-ops boundary

Use this taxonomy to answer three different evaluator questions:

- Can Topogram generate and verify software from durable intent?
- Can it guide safe change in hand-maintained code?
- Can it recover structure from real existing systems it did not design?
