# Demo Path Confirmation Plan

This note captures the current generated-app demo path.

## Canonical Demo

Use [demos/generated/todo-demo-app](../demos/generated/todo-demo-app).

## Run Sheet

```bash
cd demos/generated/todo-demo-app
npm install
npm run topogram:validate
npm run topogram:generate
npm run app:compile
```

Then run the local stack:

```bash
npm run app:bootstrap
npm run app:dev
```

In another terminal:

```bash
npm run app:smoke
npm run app:runtime-check
```

## Confirmation Criteria

- the demo uses the local engine package through `package.json`
- `topogram/` is the authored source
- `app/` is regenerated by `npm run topogram:generate`
- generated compile, smoke, and runtime-check commands are visible
- no import/adopt, maintained-app, brownfield, or `topogram-demo` step is required

## Deferred Material

Earlier maintained-app and brownfield demo paths are deferred while the generated-app workflow is tightened. Keep those materials as reference proof history, not the active demo.
