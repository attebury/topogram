## Style rules

- keep the palette warm, restrained, and proof-oriented
- prefer serif headlines and compact monospace labels
- use cards and horizontal flow, not decorative architecture maps
- preserve the exact meaning and claim boundary from the Mermaid source
- optimize for launch pages, screenshots, decks, and evaluator support material

## Working rule

If a Mermaid source diagram changes meaningfully:

1. update the Mermaid source first
2. then refresh the polished SVG variant

The polished assets should never become the place where the story changes first.
